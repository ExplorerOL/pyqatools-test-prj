## Основная информация

**PyQATools Test Project** — это тестовый проект, созданный для демонстрации использования библиотеки PyQATools в сочетании с pytest, Allure и другими инструментами для автоматизации тестирования.

- **Версия:** 0.1.0 (согласно pyproject.toml)
- **Язык программирования:** Python ≥3.10
- **Лицензия:** Не указана (предположительно проприетарная или открытая)
- **Статус:** Активная разработка

## Описание стека

Проект построен на следующих технологиях и библиотеках:

- **Python** — базовый язык.
- **pytest** — фреймворк для написания и запуска тестов.
- **PyQATools** — кастомная библиотека для улучшения процесса тестирования (логирование, репортинг, ассерты, шаги).
- **Allure-pytest** — интеграция с Allure для создания детальных отчётов.
- **pytest-xdist** — параллельный запуск тестов.
- **pytest-rerunfailures** — повторное выполнение упавших тестов.
- **pytest-check** — поддержка мягких ассертов.
- **assertpy** — библиотека для fluent-ассертов.
- **pytest-assert-that** — плагин для ассертов в стиле assertThat.
- **uv** — менеджер зависимостей и виртуальных окружений (используется через uv.lock).
- **prek** — инструмент для проверки кода перед коммитом.
- **ruff** — линтер и форматтер Python.

## Начало работы

### Предварительные требования

- Установленный Python 3.10 или выше.
- Менеджер зависимостей `uv` (рекомендуется) или `pip`.
- Git для клонирования репозитория.

### Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <URL репозитория>
   cd pyqatools-test-prj
   ```

2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   uv sync
   ```

## Основные команды и настройки

### Запуск тестов

- **Запуск всех тестов:**
  ```bash
  uv run pytest
  ```

- **Запуск тестов параллельно:**
  ```bash
  pytest -n auto
  ```

- **Запуск с повторением упавших тестов:**
  ```bash
  pytest --reruns 1 --reruns-delay 0
  ```

- **Запуск конкретного тестового модуля:**
  ```bash
  uv run pytest tests/assertions/test_assertions.py
  ```

### Генерация отчётов

Отчет формируется в директории ./reports

### Линтинг и форматирование

- **Проверка стиля кода с помощью ruff:**
  ```bash
  ruff check .
  ```

- **Автоисправление нарушений:**
  ```bash
  ruff check --fix .
  ```

- **Форматирование кода:**
  ```bash
  ruff format .
  ```

- **Запуск prek на всех файлах:**
  ```bash
  prek run --all-files
  ```

## Особенности проекта

- **Интеграция PyQATools:** В проекте активно используется кастомная библиотека PyQATools, которая предоставляет:
  - Единый контекст теста (`TestContext`).
  - Шаги ассертов (`assert_step`).
  - Мягкие ассерты (`assert_soft`).
  - Расширенное логирование и репортинг.

- **Модульная структура:** Чёткое разделение на конфигурацию, фикстуры, модели, сервисы, поддержку и тесты.

- **Allure-интеграция:** Автоматическое прикрепление логов, скриншотов (если предусмотрено) и детализация шагов теста.

- **Повторное выполнение тестов:** Настроено автоматическое повторение упавших тестов (1 раз без задержки) через pytest-rerunfailures.

- **Параллельный запуск:** Возможность ускорения выполнения с помощью pytest-xdist.

- **Использование метаклассов** Для автоматического логирования, репортинга, создания неблокрующих проверок из методов и добавления автоматических 
повторов используются наследование от базового класса шагов StepsBase. Он создан на основе метакласса, в котором происходит автоматическое применение нужных декораторов
